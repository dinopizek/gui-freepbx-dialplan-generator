FreePBX Dialplan Generator

Use Case Example

Suppose a telecom operator assigns SIM cards based on specific prefixes:

PrefixPhone NumberSIM Assignment
541098 123 1231SIM 1
542098 123 1232SIM 2
543098 123 1233SIM 3
544098 123 1234SIM 4

To generate dialplan rules for these, enter 541 as the minimum prefix and 544 as the maximum prefix, or use the range option (e.g., range 4 starting from 541). Preview and download the configuration.

Pattern Explanation

X. — Matches any single digit (0-9), repeated any number of times (e.g., 1, 12, 123...)
Z. — Matches any digit from 1-9, repeated any number of times (e.g., 1, 12, 123...)
00385Z. — Matches numbers starting with 00385 followed by one or more digits from 1-9
385Z. — Matches numbers starting with 385 followed by one or more digits from 1-9

Pattern Matching Examples

099 123 1234 — Matches X. (any number starting with a single digit, e.g., 0...)
99 123 1234 — Matches X. (any number starting with a single digit, e.g., 9...)
385 99 123 1234 — Matches 385Z. (starts with 385, then 9...)
00385 99 123 1234 — Matches 00385Z. (starts with 00385, then 9...)
123 1234 — Matches Z. (starts with 1...)

Default Settings

By default, the generator uses Croatian number format patterns:

00385Z., 385Z., X., Z., X.

These can be changed to any patterns you need for your use case. Just add, remove, or edit the patterns in the "Match Patterns" section.
